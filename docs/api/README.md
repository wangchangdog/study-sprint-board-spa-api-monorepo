# API 仕様ファイル

このディレクトリでは、機械可読な API 仕様を管理します。

## なぜ機械可読な API 仕様書が必要か

- フロントとバックで実装がずれたとき、仕様書がなければ「どちらが正しいか」を判断できない
- Markdown の文章だけでは曖昧さが残り、チームメンバー間で解釈違いが起きる
- 機械可読であれば、モックサーバー生成・クライアント生成・自動バリデーションに活用できる
- コードレビューで「仕様どおりか」を客観的に判定できる
- メンバー交代や引き継ぎ時に、口頭説明に頼らず全体像を把握できる

詳細は [docs/architecture.md](../architecture.md) の「API 仕様は OpenAPI を正本にする」を参照してください。

## この見本での方針

- **成果物の正本** は `openapi.yaml`
- `docs/api-spec.md` は教材向けの補足説明
- `TypeSpec` は将来導入してよいが、この標準見本では必須にしない

## 将来の拡張

発展課題では、以下の構成に拡張できます。

- `spec.tsp` を記述元にする
- `openapi.yaml` を生成成果物として出力する
- CI で仕様生成と差分検知を行う
